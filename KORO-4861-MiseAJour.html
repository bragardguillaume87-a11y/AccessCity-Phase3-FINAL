3. Outil de développement — Précisions techniques

3.1 Approche pédagogique en deux temps

Le dossier de candidature mentionnait GDevelop comme outil de référence pour les ateliers. L'approche pédagogique retenue conserve GDevelop dans un premier temps comme outil de découverte, avant d'utiliser un outil dédié pour la production finale :

Temps Outil Objectif pédagogique
Initiation
(séances 1–2 de chaque atelier)		 GDevelop Découvrir les notions de base du développement de jeu vidéo (scènes, événements, personnages) dans un environnement familier et documenté. Supports pédagogiques existants utilisables.
Production
(séances suivantes)		 AccessCity Studio Créer le jeu AccessCity lui-même, avec un outil conçu spécifiquement pour le roman visuel interactif et les mini-jeux de déplacement topdown — sans ligne de code.

3.2 AccessCity Studio — Présentation

AccessCity Studio est un éditeur visuel no-code développé spécifiquement pour le projet. Il combine deux modules :

  • Un éditeur de visual novel : scènes, dialogues à choix multiples, personnages avec états émotionnels, graphe narratif visuel, jet de dé intégré.
  • Un éditeur de carte 2D topdown : déplacements vue du dessus (style anciens jeux d'aventure), entités NPC avec comportements configurables, zones de déclenchement de dialogues, effets atmosphériques.
AccessCity Studio est conçu pour être utilisé directement par les participants des ateliers, y compris des personnes en situation de handicap moteur ou cognitif : interface entièrement visuelle (drag-and-drop), aucun code à écrire, annulation/rétablissement illimité, sauvegarde automatique, compatible avec les périphériques adaptés (trackball, joystick, clavier adapté). Disponible en version web (sans installation) et en application desktop Windows.

Déplacements vu en hauteur (style anciens jeux d'aventure) Module topdown 2D avec moteur de jeu intégré — déplacements au clavier, caméra suivante
Mini-jeux pédagogiques (braille, langue des signes, lecture labiale…) Scènes de jeu 2D avec zones d'interaction et déclencheurs de séquences narratives
Un personnage / un type de handicap par chapitre Système de personnages avec états émotionnels et caractéristiques configurables par personnage
Rencontres en explorant la carte → déclenchement d'un dialogue Zones de collision sur la carte 2D → ouverture automatique d'une scène VN
