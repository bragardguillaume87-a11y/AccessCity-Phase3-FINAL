# Principes fondamentaux & standards qualité

- Cohérence visuelle : respect du design system, uniformité dans la documentation et les workflows agentiques.
- Accessibilité : structure claire, audit a11y intégré, feedback utilisateur explicite, workflow utilisable par IA et humains.
- Inclusivité : langage neutre, contenu accessible à tous, onboarding facilité pour les nouveaux contributeurs.
- Documentation actionnable et IA-friendly : exemples prêts à copier, sections bien délimitées, sémantique explicite, schémas et liens vers les outils/fichiers.
- Automatisation & CI/CD : validation automatisée, intégration continue des bonnes pratiques dans les workflows.
- Contribution : feedback encouragé, documentation et workflows à jour, conventions de commit.

> Ces principes guident le workflow agentique pour garantir une expérience optimale, accélérer le développement (+20 à +30 %), réduire les bugs et faciliter l’onboarding.

# AccessCity Agentic Workflow

Ce document décrit le workflow agentique du projet AccessCity, en intégrant les meilleures pratiques open source, IA, React/Vite, CI/CD, accessibilité et gestion de communauté.

## 1. Structuration et clarté
- Chaque étape du workflow (développement, test, déploiement, rollback) est documentée.
- Les rôles (humain/IA), points de passage et critères de validation sont explicités.
- Utilisez des schémas, exemples concrets et liens vers les outils/fichiers concernés.

## 2. Automatisation et CI/CD
- Les pipelines CI/CD sont décrits (tests, build, déploiement, rollback).
- Les contributions sont vérifiées automatiquement (lint, tests, couverture).
- Liens vers les workflows GitHub Actions ou équivalents.

## 3. Accessibilité et inclusivité
- Audit d’accessibilité intégré (WCAG, outils recommandés).
- Implication d’utilisateurs réels dans l’évaluation.
- Documentation accessible (structure, navigation, contraste).

## 4. Contribution et communauté
- Entrée facilitée pour les nouveaux contributeurs (entonnoir du contributeur, issues “good first issue”).
